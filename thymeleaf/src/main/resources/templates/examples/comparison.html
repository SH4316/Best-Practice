Best vs Bad Practice 비교

Thymeleaf 사용 시 좋은 습관과 나쁜 습관을 비교하여 올바른 사용법을 익혀보세요.

주제 선택
변수 표현식 반복문 조건문 링크 처리 폼 처리 보안 성능 최적화

변수 표현식

Best Practice ✅
안전한 변수 접근
기본 이름

안전한 내비게이션 연산자(?.)를 사용하여 NullPointerException을 방지합니다.

Bad Practice ❌
위험한 변수 접근

null 체크 없이 변수에 접근하면 오류가 발생할 수 있습니다.

Best Practice ✅
유틸리티 객체 활용

Thymeleaf 유틸리티 객체를 사용하여 날짜를 포맷팅합니다.

Bad Practice ❌
하드코딩된 포맷

포맷되지 않은 날짜를 직접 출력하면 가독성이 떨어집니다.

핵심 포인트
  • 항상 안전한 내비게이션 연산자(?.)를 사용하세요
  • Thymeleaf 유틸리티 객체를 적극 활용하세요
  • null 값에 대한 기본값을 제공하세요

반복문

Best Practice ✅
상태 변수 활용

  1
  이름

상태 변수를 활용하여 인덱스, 카운트 등 추가 정보를 사용합니다.

Bad Practice ❌
상태 변수 미활용

  ID
  이름

상태 변수를 사용하지 않으면 순서, 홀짝 정보를 활용할 수 없습니다.

Best Practice ✅
빈 목록 처리

  사용자가 없습니다.


  

빈 목록을 처리하여 사용자 경험을 향상시킵니다.

Bad Practice ❌
빈 목록 처리 없음


  

빈 목록일 때 빈 테이블만 표시되어 사용자 경험이 저하됩니다.

핵심 포인트
  • 항상 상태 변수를 활용하여 풍부한 정보를 제공하세요
  • 빈 목록 처리를 통해 사용자 경험을 향상시키세요
  • 대용량 데이터는 페이징 처리를 구현하세요

조건문

Best Practice ✅
switch-case 문 활용

  관리자
  사용자
  알 수 없음

switch-case 문을 사용하여 복잡한 조건을 간결하게 표현합니다.

Bad Practice ❌
중첩 if 문

  관리자


  사용자

중복된 조건문은 코드를 복잡하게 만들고 유지보수를 어렵게 합니다.

Best Practice ✅
삼항 연산자

  상태

삼항 연산자를 사용하여 간단한 조건을 간결하게 표현합니다.

Bad Practice ❌
중복된 if 문

  상태


  상태

간단한 조건에도 중복된 코드를 사용하면 비효율적입니다.

핵심 포인트
  • switch-case 문을 사용하여 복잡한 조건을 간결하게 표현하세요
  • 삼항 연산자를 활용하여 간단한 조건을 처리하세요
  • if-unless 조합을 적절히 사용하여 코드를 단순화하세요

링크 처리

Best Practice ✅
URL 표현식 사용

  사용자 정보

Thymeleaf URL 표현식을 사용하면 컨텍스트 경로를 자동으로 처리합니다.

Bad Practice ❌
하드코딩된 URL

  사용자 정보

하드코딩된 URL은 컨텍스트 경로 변경 시 문제가 발생합니다.

Best Practice ✅
정적 리소스 처리

정적 리소스도 URL 표현식을 사용하여 일관성을 유지합니다.

Bad Practice ❌
상대 경로 사용

상대 경로는 파일 위치 변경 시 링크가 깨질 수 있습니다.

핵심 포인트
  • 항상 URL 표현식(@{...})을 사용하여 동적 URL을 생성하세요
  • 정적 리소스도 URL 표현식을 사용하여 일관성을 유지하세요
  • 쿼리 파라미터는 URL 표현식 내에서 처리하세요

폼 처리

Best Practice ✅
th:object와 th:field 사용

  
  

    
  

  

th:object와 th:field를 사용하면 폼 바인딩과 유효성 검사가 자동화됩니다.

Bad Practice ❌
수동 폼 처리

  
  
  
  

수동 폼 처리는 코드가 길어지고 오류가 발생하기 쉽습니다.

핵심 포인트
  • th:object와 th:field를 사용하여 폼 바인딩을 자동화하세요
  • 유효성 검사를 구현하여 데이터 무결성을 보장하세요
  • Spring Security를 사용하면 CSRF 토큰이 자동으로 추가됩니다

보안

Best Practice ✅
XSS 방지
댓글

th:text를 사용하면 자동으로 HTML 이스케이프가 적용되어 XSS를 방지합니다.

Bad Practice ❌
XSS 취약점
댓글

th:utext를 사용하면 악성 스크립트가 실행될 수 있습니다.

Best Practice ✅
권한 체크

  

권한 체크를 통해 인가된 사용자만 민감한 기능에 접근하게 합니다.

Bad Practice ❌
권한 체크 없음

권한 체크 없이 민감한 기능을 노출하면 보안 위험이 발생합니다.

보안 핵심 포인트
  • 항상 th:text를 사용하여 자동 HTML 이스케이프를 적용하세요
  • th:utext는 신중하게 사용하고, 반드시 이스케이프 처리를 하세요
  • 권한 체크를 통해 인가된 사용자만 민감한 기능에 접근하게 하세요
  • CSRF 토큰을 사용하여 CSRF 공격을 방지하세요

성능 최적화

Best Practice ✅
불필요한 연산 방지

  날짜
  날짜

th:with를 사용하여 반복적인 연산을 한 번만 수행합니다.

Bad Practice ❌
반복적인 연산

동일한 연산을 반복하면 성능이 저하됩니다.

Best Practice ✅
지연 로딩

  
    
      
    
  

필요한 시점에 데이터를 로드하여 불필요한 연산을 방지합니다.

Bad Practice ❌
대용량 데이터 전체 로딩


  

대용량 데이터를 한 번에 로드하면 메모리 문제가 발생할 수 있습니다.

성능 최적화 핵심 포인트
  • th:with를 사용하여 반복적인 연산을 방지하세요
  • 필요한 시점에 데이터를 로드하여 지연 로딩을 구현하세요
  • 대용량 데이터는 페이징 처리를 구현하세요
  • 운영 환경에서는 템플릿 캐싱을 활성화하세요
